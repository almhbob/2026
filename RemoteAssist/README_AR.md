# RemoteAssist — تطبيق دعم فني عن بُعد بين هاتفين

هذا مشروع MVP آمن لتطبيق يشبه AnyDesk على أندرويد، لكنه مبني على الموافقة الواضحة من صاحب الهاتف المضيف:

- الهاتف الأول: يختار **مشاركة هاتفي** ويوافق على نافذة Android الرسمية لتسجيل/التقاط الشاشة.
- الهاتف الثاني: يختار **مشاهدة/تحكم** ويدخل رمز الجلسة الظاهر في الهاتف الأول.
- التحكم باللمس على الهاتف المضيف لا يعمل إلا بعد أن يفعّل صاحب الهاتف خدمة Accessibility يدويًا من إعدادات أندرويد.
- توجد خدمة Foreground Notification أثناء المشاركة ولا توجد أي طريقة خفية للتشغيل.

> هذا التصميم مخصص للدعم الفني المصرّح به فقط، وليس للمراقبة أو التحكم دون علم صاحب الهاتف.

## محتويات المشروع

```text
RemoteAssist/
├─ android/   تطبيق Android Kotlin Native
└─ server/    خادم WebSocket بسيط للربط بين الهاتفين
```

## التشغيل السريع

### 1) تشغيل السيرفر

على جهاز/سيرفر Node.js:

```bash
cd server
npm install
npm start
```

سيعمل السيرفر افتراضيًا على:

```text
ws://YOUR_SERVER_IP:8080
```

للتجربة على نفس شبكة Wi‑Fi، استخدم IP جهاز الكمبيوتر بدل `YOUR_SERVER_IP`.

### 2) فتح تطبيق أندرويد

افتح مجلد `android` في Android Studio ثم شغّل التطبيق على هاتفين أندرويد.

أو من الطرفية إن كان لديك Android SDK و Gradle:

```bash
cd android
gradle :app:assembleDebug
```

سيظهر ملف APK داخل:

```text
android/app/build/outputs/apk/debug/app-debug.apk
```

### 3) طريقة الاستخدام

1. في الهاتف المضيف: أدخل رابط السيرفر مثل `ws://192.168.1.10:8080`.
2. اضغط **مشاركة هاتفي**.
3. وافق على نافذة Android الرسمية لمشاركة الشاشة.
4. فعّل زر **تفعيل التحكم اللمسي** وافتح إعدادات Accessibility، ثم فعّل `RemoteAssist Control` إن أردت التحكم وليس المشاهدة فقط.
5. في الهاتف الآخر: أدخل نفس رابط السيرفر ورمز الجلسة، ثم اضغط **مشاهدة/تحكم**.
6. اضغط على الشاشة المعروضة لإرسال Tap للهاتف المضيف.

## ملاحظات تقنية مهمة

- بث الشاشة في هذا MVP يتم كصور JPEG متتالية عبر WebSocket، مناسب للاختبار والدعم الخفيف.
- للإصدار التجاري القوي، استبدل نقل الصور بـ WebRTC + H.264/VP8 لتقليل التأخير ورفع الجودة.
- أندرويد لا يسمح بالتقاط الشاشة دون موافقة المستخدم، ولا يسمح بحقن اللمسات إلا بخدمة Accessibility مفعّلة يدويًا.
- التطبيق لا يحتوي على Boot Receiver ولا يعمل بعد إعادة التشغيل تلقائيًا.
- لا توجد وظائف مخفية، ولا التقاط بيانات، ولا تحكم دون إشعار.
